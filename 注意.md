# Role
你是一个精通 Java Spring Boot、Vue.js 和 LLM 应用开发的资深全栈架构师。

# Project Goal
我要开发一个基于 Java 的“智能运维辅助 Agent (AIOps Agent)”。
核心功能是：后端定时或主动通过 SSH 抓取服务器日志 -> 交给文心一言 (通过 LangChain4j) 进行分析 -> 提取报错和建议 -> 前端展示为可视化卡片 -> 用户点击按钮即可自动执行修复命令。

# Tech Stack
- **Backend**: Java 17+, Spring Boot 3.x
- **AI Integration**: LangChain4j (langchain4j-qianfan 用于对接文心一言)
- **SSH Tool**: JSch (用于连接 Linux 服务器执行命令)
- **Frontend**: Vue 3 + Element Plus (或其他简单的 UI 库)
- **Data Exchange**: REST API, JSON

# Core Workflow (必须严格遵守)
1. **Log Fetching (Agentless)**: 后端提供工具类，通过 SSH 连接目标服务器 (默认 Ubuntu 22)，使用 `tail -n 50` 或 `grep` 获取 Nginx/Tomcat/System 的日志片段。
2. **AI Analysis**: 将日志片段发送给文心一言。
   - **重点**: 系统提示词必须强制 AI 返回纯 JSON 格式。
   - JSON 结构需包含：错误摘要(summary)、技术分析(analysis)、解决方案列表(actions)。
   - 解决方案列表中的每个动作需包含：按钮文字(label)、具体 Shell 命令(command)、风险等级(risk)。
3. **Dashboard UI**: 前端**不是**聊天窗口。是一个“运维监控大屏”。
   - 界面展示服务器列表。
   - 展示 AI 分析出的“异常卡片”。
   - 卡片上渲染“修复按钮”。
4. **Execution**: 用户点击前端按钮 -> 调用后端 API -> 后端通过 SSH 执行该按钮对应的 command -> 返回执行结果。
